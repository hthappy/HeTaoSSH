# 终端分屏清空内容问题 - 总结与解决方案

## 问题描述

使用快捷键分屏（Ctrl+Shift+D 横向，Ctrl+Shift+E 纵向）时，**原终端的内容被完全清空**，只剩下空白屏幕和光标。

## 根本原因（✅ 100% 确认）

通过调试日志确认：**Terminal 组件在分屏时被 React 卸载并重新挂载**

```
[Terminal] Creating terminal instance, serverId: 7
[Terminal] DISPOSING terminal instance, serverId: 7  ← 组件被销毁
[Terminal] Creating terminal instance, serverId: 7  ← 重新创建
```

当组件被卸载时，xterm.js 实例的 `dispose()` 方法被调用，导致：
- 终端缓冲区被清空
- 滚动历史丢失
- 屏幕内容消失

### 为什么组件会被卸载？

**专家解答**：React 的 `key` 仅在**父节点相同的同级兄弟元素之间**进行对比时才生效。

当触发分屏时：
1. 原本的 `<SingleTerminal />` 被重新包裹进了 `<SplitPane>` 结构中
2. 由于**父节点（上下文）变了**，React 会毫不留情地将这棵子树连根拔起（Unmount）
3. 从而触发了 `term.dispose()`，导致缓冲区彻底清空

## 最终解决方案：DOM 节点物理转移 (DOM Reparenting) ✅

### 核心思想

**不在 React 组件中调用 term.open()**。我们把 xterm.js 及其挂载的 `<div>` 完全移出 React 的管理范畴，React 组件只负责提供一个"占位坑"（Placeholder）。

### 实现步骤

#### 1. 创建全局的 Terminal 缓存池 (`web/src/lib/terminalPool.ts`)

脱离 React，用一个普通的 TypeScript Map 来保存真正的 DOM 节点和 xterm 实例：

```typescript
class TerminalPool {
  private instances = new Map<string, TerminalInstance>();

  getOrCreate(paneId: string, config: {...}): TerminalInstance {
    // 创建 xterm 实例 + DOM 容器（只创建一次）
    // 存储在 Map 中，以 paneId 为键
    const container = document.createElement('div');
    const term = new XTerm({ /* config */ });
    term.open(container);
    return { term, container, fitAddon, searchAddon };
  }

  dispose(paneId: string): void {
    // 只在关闭标签时调用
    // 调用 term.dispose() 并从 Map 中移除
  }
}

export const terminalPool = new TerminalPool();
```

#### 2. 改造 React 组件 (`web/src/components/Terminal.tsx`)

在 React 组件中，使用 useRef 获取占位符，然后利用原生 DOM API 把缓存的节点"塞"进去：

```typescript
export const Terminal = forwardRef<TerminalHandle, TerminalProps>(
  function Terminal({ paneId, ...props }, ref) {
    // CRITICAL: 这只是一个占位符 div，不是真正的终端容器
    const placeholderRef = useRef<HTMLDivElement>(null);
    
    useEffect(() => {
      if (!placeholderRef.current) return;

      // 1. 从池中获取或创建实例
      const instance = terminalPool.getOrCreate(paneId, { fontSize, lineHeight });
      
      // 2. 使用原生 DOM API 附加容器
      placeholderRef.current.appendChild(instance.container);
      
      // 3. 设置事件处理器
      const onDataDisposable = instance.term.onData((data) => {
        // 处理数据
      });
      
      // 4. Cleanup: 只移除 DOM 节点，不 dispose
      return () => {
        if (placeholderRef.current && instance.container.parentNode === placeholderRef.current) {
          placeholderRef.current.removeChild(instance.container);
        }
        onDataDisposable.dispose();
      };
    }, [paneId]);

    return <div ref={placeholderRef} className="..." />;
  }
);
```

#### 3. 更新 TerminalArea 组件 (`web/src/components/TerminalArea.tsx`)

传递 `paneId` 属性：

```typescript
<TerminalComponent
  paneId={pane.id}  // REQUIRED
  onData={handleTerminalData}
  // ... 其他属性
/>
```

#### 4. 在关闭标签时清理 (`web/src/stores/ssh-store.ts`)

```typescript
closeTab: (tabId: string) => {
  // 释放所有终端实例
  const paneGroup = get().paneGroups[tabId];
  if (paneGroup) {
    const disposePanes = (group: PaneGroup) => {
      for (const pane of group.panes) {
        if ('serverId' in pane) {
          terminalPool.dispose(pane.id);
        } else if ('direction' in pane) {
          disposePanes(pane);
        }
      }
    };
    disposePanes(paneGroup);
  }
  // ... 其他清理逻辑
}
```

### 为什么这个方案能完美解决问题？

#### 对抗 React 的强制卸载

当用户按下 `Ctrl+Shift+D` 触发分屏时，React 确实会把原本的 `<TerminalComponent />` 卸载掉。

#### 保护机制

在这个方案中，React 卸载组件时触发的 useEffect cleanup 仅仅执行了 `removeChild(container)`。

**xterm.js 实例完全不知道自己被移出了文档流**，它的内存缓冲区、滚动历史全都在。

#### 瞬间重组

零点几毫秒后，React 在 `<SplitPane>` 的左侧渲染了一个新的 `<TerminalComponent />`。

useEffect 触发，把那个原封不动的 container 重新 appendChild 进去，**内容瞬间恢复，连光标位置都不会变**。

## 与之前修复的区别

### 标签切换黑屏（✅ 已修复 - 2024-03-23）
- **问题**：切换标签时终端内容消失
- **原因**：标签隐藏时使用 `display: none`，渲染器未准备好
- **解决**：使用 `visibility: hidden`，等待渲染器初始化
- **关键**：组件**没有被卸载**，只是隐藏/显示

### 分屏清空内容（✅ 已修复 - 2024-04-10）
- **问题**：分屏时终端内容消失
- **原因**：组件树重组导致组件**被卸载**
- **解决**：DOM Reparenting 模式，将 xterm.js 实例移出 React 生命周期
- **关键**：使用原生 DOM API 物理移动容器，而不是让 React 管理

## 技术洞察

1. **React 的 key 只对同级兄弟有效**：当分屏将组件包装在 `<SplitPane>` 中时，父节点改变，导致卸载
2. **xterm.js 是命令式的**：无法由 React 声明式管理
3. **VS Code 的解决方案**：将终端实例保存在全局池中，使用 DOM API 移动容器
4. **关键**：`appendChild`/`removeChild` 物理移动 DOM 节点而不销毁它们

## 核心矛盾（已解决）

### React 的声明式模型
```
状态变化 → 虚拟 DOM 对比 → 协调 → DOM 更新
```
React 决定何时复用/卸载组件

### xterm.js 的命令式模型
```
new XTerm() → term.open() → term.write() → term.dispose()
```
xterm.js 管理终端状态（缓冲区、光标、内容）

### 解决方案
**将 xterm.js 完全移出 React 的管理**，使用原生 DOM API 进行物理移动，React 只管理占位符。
